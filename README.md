Tijdens het verwerken van feedback ben ik bewuster gaan nadenken over UX en layout:

- Mobile-first ontwerp met progressive enhancement

- Velden uitgelijnd in een grid-layout voor visuele rust

### Hoeveel tijd heeft me dat gekost?
De hele dag gewerkt met een uur pauze.  
Weekly Geek: ±1 uur.

### Wat heb ik geleerd?
Ik wilde legends eerst met `float` positioneren, maar dat gaf veel layoutproblemen.  
Dit heb ik opgelost door ze een achtergrond te geven over de volle breedte van de fieldset.

Ook heb ik geleerd dat padding op parent-elementen beter werkt dan losse margins voor consistente layouts.

Ook heb ik geleerd dat CSS zoals `:has` niet alleen styling kan doen, maar ook logica kan sturen.

Kleine keuzes zoals uitlijning en maximale breedte maken een groot verschil in hoe een interface aanvoelt.

# Ontwerpkeuzes

Tijdens dit project heb ik bewust gekozen voor:

- Mobile-first ontwerp
- Een maximale breedte van ±800px voor leesbaarheid
- Grid-layout voor consistente uitlijning
- Fieldsets voor duidelijke structuur

Deze keuzes zorgen ervoor dat het formulier niet alleen functioneel is, maar ook rust uitstraalt, iets dat belangrijk is bij een belastingformulier.
